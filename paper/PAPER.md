# Reproducing the paper

This document maps each numbered claim, figure, and table in the paper to the command
that regenerates it. Every result comes from the runnable `private-ai-platform-kit`
gateway and the harness under `paper/`. Nothing here needs a GPU.

The evaluated revision is tagged `v1.1.0-paper` in
<https://github.com/RamazanKara/private-ai-platform-kit>; check out that tag to
reproduce the numbers in the paper.

## Prerequisites

- The `private-ai-platform-kit` checkout, with the inference-gateway virtualenv built
  (`src/inference-gateway/.venv`), which provides FastAPI, uvicorn, httpx, and pydantic.
- Python 3.12+ for the harness. `matplotlib` is needed only to render the figures.
- Set `GATEWAY_DIR` to the gateway directory if it is not at the default path used by
  the scripts.
- The external baseline needs LiteLLM in its own environment (`pip install 'litellm[proxy]'`);
  point the runner at it with `LITELLM_BIN=/path/to/venv/bin/litellm`.

The harness drives the gateway as an external process, so it measures the real
service rather than a reimplementation.

## End-to-end measurement

```bash
# 1. Run the full matrix (baseline, ungoverned, governed; sweeps concurrency and
#    backend think-time; 5 repeats per cell, 2 s warmup discarded, 8 s measured).
GATEWAY_DIR=/path/to/src/inference-gateway \
  /path/to/.venv/bin/python paper/cost-of-compliance/run_experiment.py

# 2. Aggregate per-cell statistics and print the numbers the paper cites.
python paper/cost-of-compliance/analyze.py

# 3. Render the figures (needs matplotlib).
python paper/cost-of-compliance/figures.py
```

`run_experiment.py` writes `results/raw-results.json`; `analyze.py` writes
`results/aggregate.json` and `results/paper-numbers.json`. A `SMOKE=1` environment
variable runs a fast subset for a sanity check.

## Method notes

- The mock runtime (`mock_runtime.py`) is a raw asyncio HTTP/1.1 server. It sets
  `TCP_NODELAY` and `TCP_QUICKACK` so that the loopback Nagle / delayed-ACK
  interaction does not inject a per-request stall unrelated to the gateway. An
  optional fixed think-time stands in for inference time.
- Load is generated by multiple `bench.py` client processes so that the load
  generator does not become the bottleneck; latencies are pooled and throughput
  summed across shards.
- The microbenchmark removes the network entirely and times the gateway's real
  governance functions, so it is the clean per-request CPU figure. The end-to-end
  numbers include ordinary proxy I/O, which is why the governed-minus-ungoverned
  delta, not the absolute number, is the cost of governance.
